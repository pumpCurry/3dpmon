# ADR-0005 フィラメント切れ/交換の状態認識つき帰属モデル

## ステータス
提案（設計承認済み: ユーザー 2026-06-06）。**実装は次セッション**（要注意領域のため実装→単体テスト→実機検証→デプロイの一巡が必要。ADR-0004 を前提に拡張）。

## コンテキスト
ADR-0004 で残量を mountHistory 台帳＋冪等 reconcile に再設計したが、印刷途中のスプール交換／フィラメント切れ時の帰属に複数の不具合と仕様未定義があることが、対話と再現で判明した。

### 物理モデル（ユーザー確定）
- **稼働中（state=printing, 一時停止でない）に物理スプール交換は不可**（巻かれた途中フィラメントを抜けない）。→ 稼働中に登録された交換/セットは「**その印刷の開始前に装着していた**」とみなし、**ジョブ全体をその1スプールに帰属**（旧スプールの当該ジョブ debit は遡及取消）。
- **一時停止中（state=paused）はフィラメント切れ→物理交換が実在する**。→ これは「**1ジョブに複数リールを供した**」＝分割帰属（旧→切れ時点まで、新→再開後）。`filamentInfo[]` 複数スプールはこの文脈で必須（S1の結論：複数リール/ジョブはスコープ内）。
- フィラメント切れセンサーは誤動作しうる（2秒バッファでON判定）。詰まり/ゴミ取りで**同一リールを戻す**ことも多く、必ず新リールとは限らない。

## 決定：状態認識つき帰属モデル

イベント（切れ/交換）発生時の**印刷状態と時刻を記録**し、その時点に**遡及**して帰属する。

### 判定表
| 発生時の状態 | ユーザー操作 | 帰属 |
|---|---|---|
| 稼働中(printing, not paused) | 後から指示/登録 | **ジョブ全体→その1スプール**（印刷前装着扱い・遡及）。旧スプールの当該ジョブ debit を取消 |
| 一時停止中(paused) | リール指示 | **分割（複数リール）**：旧→切れ時点まで（切れ確定なら0へ駆動）、新→再開後の実測分 |
| 一時停止中 | 無視、旧スプール**残<10%**（センサーON＋推定が一致） | **暫定推定**：同プリセットの新品(満タン仮定)を投入。旧→0、新→残り。`inferred:true` フラグ＋ユーザー訂正可 |
| 一時停止中 | 無視、旧スプール**残≥10%**（不一致） | **同一スプール再セット**（新規作成なし・分割なし）。詰まり/誤動作想定 |
| 誤動作で復帰のみ | キャンセル/放置→既定 | 無変化（同一継続） |

### 暫定推定ルール（R1〜R5・承認済み）
- **R1 在庫汚染防止（最重要）**：自動生成スプールは **`inferred:true`（暫定・ユーザー確定待ち）**。silent な実在庫化はしない。確定/差し替え/取消が可能。`spoolSerialCounter` の不可逆消費を避ける。
- **R2 満タン仮定は推定値**：新スプール開始残量は満タン仮定だが推定として明示（半端継ぎ足しは訂正可）。再開後の使用量は usedMaterialLength 実測。
- **R3 ≥10% は同一再セット**（新規作成しない）。
- **R4 遡及起点と発火時点**：帰属の起点は**ランナウト/一時停止イベントの時点**（クリック時ではない）。「無視」確定は再開時/完了時。
- **R5 状態・時刻の記録**：イベント発生時の state と時刻を per-host に記録し、後からの操作でもイベント時点に遡及して稼働中(全体)／一時停止(分割)を判定。
- **<10% の意味**：単独閾値ではなく「センサーON（空）」と「推定残<10%（少）」の**2信号一致ゲート**。両者一致時のみ「使い切り→新リール」を高確度推定。不一致は詰まり/誤動作扱い（R3）。切れ確定時、旧スプールは物理的に空なので0へ駆動するのが正。

## 実装計画（次セッション・段階的）
mountHistory 台帳ベースで、すべて冪等・実テスト付き。

- **P1 状態認識の基盤**：per-host に「切れ/一時停止イベント（state, ts, 旧残%, runout有無）」を記録。`setCurrentSpoolId` / 交換ウインドウ確定時に、このイベント文脈を参照。
- **P2 稼働中=ジョブ全体（B2の稼働中側）**：稼働中登録では旧スプールを finalize/分割せず、新スプール mount を **sinceJobId < J**（＝J全体を新区間へ）にし、旧スプールの J 分は計上しない。filamentInfo に分割を書かない。
- **P3 一時停止中=分割（B2の一時停止側を正として維持・整備）**：旧→切れ時点まで（確定なら0）、新→再開後。filamentInfo[] で2リール記録。
- **P4 B1 ライブ再ベースライン**：スプール変化検知時に `s.accumulatedUsedMaterial`/`prevUsedMaterialLength`/`currentJobStartLength` を**文脈に応じて**再設定（稼働中=新スプールのジョブ開始前残量基点＋ジョブ総使用量、一時停止=新スプール現残量基点＋再開後分）。0張り付き解消。
- **P5 ウインドウ解決（B3/B4/B5）**：切れ復帰(1→0)でウインドウ自動クローズ＝既定キャンセル＝同一継続。キャンセルを「同一スプール継続」と明示。放置の既定＝同一継続（タイムアウト or 復帰時）。
- **P6 暫定推定（R1〜R5）**：一時停止中・無視・<10%（2信号一致）で `inferred` スプール生成＋分割記録。UIで「推定」表示＋訂正導線。
- **P7 reconcile 整合**：unmount 追記前後で reconcile 結果が揺れる問題（Z4）を解消（イベント確定をアトミックに）。

## テスト計画
- 単体（ledger/spool/aggregator をモックで駆動）：稼働中交換=全体帰属、一時停止交換=分割、B1再ベースライン（0に張り付かない）、暫定推定の生成/2信号ゲート/≥10%は非生成、切れ復帰でウインドウ既定解決、冪等性。
- 回帰：本対話で作成した「途中交換シミュレーション」を正式テスト化。
- 実機：2機で 稼働中登録・一時停止交換・無視<10% を実演し残量とfilamentInfoを検証（要リビルド＝終了の事前確認）。

## リスク/留意
- 自動スプール生成は**過去のフィラメント整合性危機**の再来源になりうる→必ず `inferred`＋訂正可（R1）。
- 状態判定は K1 の state 報告（printing=1/paused=5）に依存。pause検出が機器仕様で揺れる場合のフォールバックを用意。
