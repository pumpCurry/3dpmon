# ADR-0002 GridStack パネルシステムの採用

## ステータス
採用・実装済み (v2.1)

## コンテキスト
v2 初期計画では Vite + コンポーネント分離 (Card ベース) で UI を構築する予定だった。しかし以下の課題により方針転換した:

- Vite ビルドシステムの導入が既存の単一 HTML 構造と相容れない
- Card クラスベースのコンポーネント化は過剰な抽象化
- ユーザーが自由にパネルを配置・リサイズしたい要求

## 決定
GridStack.js を採用し、既存の HTML テンプレートをクローンするパネルシステムを構築する。

- `dashboard_panel_factory.js`: パネル CRUD、テンプレートクローン、スコープID変換
- `dashboard_panel_boot.js`: HTML構造からテンプレート抽出、初回レイアウト構築
- `dashboard_panel_menu.js`: サイドメニューからパネル追加
- `dashboard_panel_init.js`: パネル初期化/破棄レジストリ

### 主要な設計判断
1. **テンプレートクローン方式**: 既存の `3dp_monitor.html` 内のDOM構造をそのまま流用
2. **スコープID**: `{hostname}__originalId` でホスト別に一意な要素IDを生成
3. **レイアウト永続化**: GridStack の座標情報を localStorage に保存
4. **パネル種別 × ホスト**: 同一パネル種別を複数ホスト分生成可能

## 結果
- 既存の HTML/CSS を大幅書き換えせずにパネル化を実現
- ユーザーがドラッグ&リサイズでレイアウトを自由にカスタマイズ可能
- per-host パネル生成により、マルチプリンタの並行監視に対応
- Vite/ビルドツール不要で、HTTP サーバーだけで動作
