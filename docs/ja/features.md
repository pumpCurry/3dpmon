# ダッシュボード機能一覧

このドキュメントでは3dpmonダッシュボードの各部を説明し、
操作できる内容や管理機能をより詳しく紹介します。

## パネルシステム (GridStack)
GridStack ベースのパネルシステムにより、ダッシュボードのレイアウトを自由にカスタマイズできます。

- **ドラッグ&ドロップ**: パネルのタイトルバーをドラッグして任意の位置に配置
- **リサイズ**: パネルの端をドラッグしてサイズを変更
- **パネル追加メニュー**: メニューから必要なパネルを選んで追加。同じ種類のパネルを複数配置可能
- **レイアウト永続化**: 配置やサイズは自動保存され、次回起動時に復元
- **per-host パネル**: 各パネルは特定のプリンタに紐付き、異なるプリンタのパネルを並べて比較可能

## マルチプリンタ対応
複数台の 3D プリンタを同時に監視できます。

- **接続管理モーダル**: プリンタの追加・接続・切断を一元管理
- **per-host データ分離**: 各プリンタの温度・状態・ファイルリスト・印刷履歴はすべて独立
- **per-host カメラ ON/OFF**: プリンタごとにカメラの有効/無効を個別設定
- **per-host フィラメント装着**: `hostSpoolMap` によりプリンタごとに異なるスプールを装着
- **per-host TTS 設定**: プリンタごとに異なる TTS 音声・速度を設定可能
- **独立した WebSocket 接続**: 各プリンタへの接続はホストごとに独立管理、自動再接続対応

## Electron アプリ
ブラウザ版に加え、Electron によるデスクトップアプリとして利用できます。

- `npm run electron` で起動
- ブラウザ版と同一機能、HTTP サーバー不要
- ブラウザ版も引き続きサポート (`start.bat` またはHTTPサーバー経由)

## ヘッダ
接続管理ボタンを備え、タイトル部にはホスト名と接続状態を表示します。
再接続は自動で試行されます。

## 監視
ライブカメラ映像と状態表示をまとめ、印刷の一時停止・再開・停止が行えます。
ノズルやベッド温度の調整も可能です。

## ヘッドプレビュー
ヘッドの位置をリアルタイムで示し、プリンタのフタを開けずに
動作状況を確認できます。回転状態はプリンタごとに独立して保持されます。

## フィラメントプレビュー
残量を3Dスプール図で表示し、印刷完了ごとに自動更新されます。
残りが少なくなると警告を出します。プリンタごとに装着中のスプールが異なる場合も正しく表示します。

## 状態
進捗、経過時間、残り時間に加え、予想フィラメント使用量を表示します。

## 操作パネル
モデル・ケース・補助ファンやLEDの切り替え、印刷操作ボタンを備えています。
スライダーでフィードレートやフロー率、温度を変更できます。
コマンドは接続先のプリンタに対して送信されます。

## 温度モニタ
ノズルとベッドの温度推移をグラフ化し、過去データも確認できます。
各プリンタのグラフは独立したバッファで管理されます。

## 機器情報
各軸の制限値やファームウェア情報などのハードウェア詳細を確認できます。

## ログ
送受信したメッセージと通知履歴を表示し、ログのコピーも可能です。

## 現在の印刷
印刷中ファイルの概要と選択中のフィラメント情報を表示します。

## 印刷履歴
完了したジョブの一覧をサムネイル付きで表示し、録画動画へのリンクも付きます。

## ファイル一覧
G-codeのアップロード、削除、実行が可能です。ここから直接印刷開始できます。
ファイルリストはプリンタごとに独立管理されます。

## 設定

### ストレージ設定
IndexedDB（優先）または localStorage でデータを管理します。
v2.00 形式での全データエクスポートと、v1.40/v2.00 両形式に対応したインポートが行えます。

### 通知設定
通知設定は **接続管理モーダルのサブモーダル** に移動しました。
プリンタごとに以下を設定できます:

- 効果音・音声通知の有効/無効
- 異常温度の警告閾値
- カテゴリ別通知の有効/無効 (印刷イベント / カメラ / 温度アラート)
- TTS 音声の種類と速度 (per-host 設定)

### フィラメントスプール
スプールの登録・編集・削除が可能で、残量は自動追跡され不足時に警告します。
マルチプリンタ環境では `hostSpoolMap` により各プリンタに正しくスプール消費が紐付きます。

### コマンドパレット
原点復帰、ベッドレベリング、ファームウェア更新、工場出荷状態へのリセット、
任意JSON送信などのコマンドをまとめています。カスタムG-codeも送信できます。
コマンドはパネルに紐付くプリンタに対して送信されます。
