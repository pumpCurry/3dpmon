# フィラメントスプール管理と在庫追跡

ここでは、3dpmon (v2.1.004) のフィラメント管理機能を活用してスプール残量を把握し、在庫を管理する方法を紹介します。

## スプール登録
- フィラメント管理パネルの **追加** ボタンからスプール情報を登録します。
- スプール名、総巻き長さ、現在の残り長さに加え、メーカー名や材料名も設定可能です。
- 登録されたスプールには **#NNN** 形式のシリアル番号が自動付与され、一覧での識別が容易になります。
- 登録後はパネル上に 3D スプールプレビューが表示され、消費量が自動的に更新されます。

## グローバルデータと Per-host データの区別
v2.1.004 では、フィラメント関連データはグローバルと per-host に分離管理されます。

| データ種別 | 管理範囲 | 説明 |
|---|---|---|
| スプールプリセット | グローバル | 材料・メーカー等のプリセット定義。全プリンタ共通 |
| スプール在庫 | グローバル | スプールリスト・在庫数。どのプリンタからでも同じ在庫を参照 |
| 装着スプール | **Per-host** | `hostSpoolMap` で各プリンタに現在装着中のスプールを個別管理 |
| フィラメント消費量 | **Per-host** | 消費履歴はプリンタごとに記録。データの混在なし |

## Per-host スプール装着
- **`hostSpoolMap`** により、各プリンタにどのスプールが装着されているかを個別に管理します。
- プリンタ A に PLA スプール、プリンタ B に ABS スプールを装着するなど、独立した運用が可能です。
- フィラメント交換操作（`setCurrentSpoolId` / `useFilament` / `reserveFilament` / `finalizeFilamentUsage`）はすべて hostname 引数付きで呼び出され、正しいプリンタに紐付きます。

## 在庫管理
- スプール交換時に在庫数が自動で減算されるため、手動入力の手間が省けます。
- 在庫が不足すると警告が表示され、購入リンクを開くこともできます。

## Per-host フィラメント消費追跡
- 消費履歴はプリンタごとに個別記録されます。
- 集計レポート機能を使うと、日別や月別の消費量をグラフで確認できます。
- どのプリンタでどれだけのフィラメントを消費したかを正確に把握できます。

## パネルシステム
フィラメント管理は GridStack パネルとして表示されます。3D スプールプレビューや消費量グラフをドラッグ・リサイズで自由に配置できます。

## 活用のヒント
- `dashboard_spool.js` を改造することで、独自のスプールデータベースを構築できます。
- 集計レポート機能を使うと、プリンタ別・日別・月別の消費量をグラフで確認できます。
